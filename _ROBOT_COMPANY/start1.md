# серьёзный уровень — closed-loop система с абсолютным позиционированием.

# Стерео-камеры (2 камеры)
Определяют 3D-положение рук в пространстве (реальные координаты X,Y,Z).
Программа сравнивает это с forward kinematics (расчёт позиции рук по углам всех моторов).
Получается коррекция ошибок (механический люфт, прогиб рычагов, неточность редукторов и т.д.).

# Знание позиций моторов
Каждый ESP32 постоянно шлёт актуальные позиции (в тиках холлов) всех моторов.
C# хранит внутреннюю модель робота (kinematic chain).

# Лазерный дальномер (VL53L1X, TFmini-S, или несколько штук)
Даёт точное расстояние до груши + можно сделать сканирование для определения центра груши.

# C# программа
Получает данные по WiFi (UDP).
Строит 3D-сцену: положение робота + положение груши + реальное положение рук.
Планирует удар (траекторию, силу, угол).
Отправляет целевые позиции/скорости/торк на ESP32.

# Пакет от C# → ESP32 (команды):
Целевые позиции для каждого мотора (или группы)
Режим (position / velocity / torque)
Параметры удара (сила, тип: прямой, хук, апперкот и т.д.)

# Одна ESP32 (Motor Controller) будет делать следующее:
Принимать UDP от C# программы (команды: целевые позиции, удары и т.д.)
Раздавать команды по UART на две платы гироскутера
Читать текущие позиции/токи с двух плат
Формировать пакет телеметрии

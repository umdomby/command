Stack - C# Net9.0; Next16; Postgres Prisma; Nginx; Docker - server Ubuntu

JoyControl — это программа для Windows, предназначенная для настройки и эмуляции контроллеров. Похожие программы: X360CE и DS4Windows, но имеющий уникальную
функциональность, имеет встроенную систему лицензий и оптовые продажи ключей доступа.

Триал: 1 час в день бесплатно.
На год: Полный доступ на 1 год 15$.
На два года: Полный доступ на 2 года 20$.
Пожизненная: Полный доступ навсегда 50$.

Сайт программы https://ardu.live/joy/control
Программа написана на C# Net9.0 используются библиотеки HidSharp, SharpDX.XInput,
SharpDX.DirectInput, Nefarius.ViGEm.Client

Для получения триал и другой лицензии нужно зарегистрироваться на сайте (Next16) https://ardu.live/register  после оплаты администратор активирует нужную лицензию.

######     Жизненный цикл лицензии      ######

Пользователь впервые запускает программу
→ вводит email + пароль
→ приложение отправляет их на /api/joy/login
Сервер отвечает:Вариант А — пользователь новый → создаёт триал на 60 минут в сутки
Вариант Б — пользователь уже есть → проверяет пароль
→ проверяет, не занято ли это устройство (HWID) другим аккаунтом
→ если всё ок → добавляет HWID в список разрешённых (максимум 3 за 7 дней)
→ выдаёт токен (это как временный пропуск)
Приложение сохраняет токен зашифрованно в файл login.dat
Каждый раз при запуске (и периодически во время работы) приложение делает:POST /api/joy/validate
(передаёт токен + deviceId)Сервер отвечает примерно так:JSON{
"valid": true,
"type": "trial",           // или "monthly", "yearly" и т.д.
"trialRemaining": 43,
"trialLimit": 60,
"message": "Trial mode active"
}Если токен недействителен / просрочен / HWID не разрешён → сервер говорит «нет» → приложение переходит в оффлайн-режим по последнему известному состоянию.
Когда пользователь пользуется программой → приложение каждые N минут отправляет heartbeatPOST /api/joy/heartbeat
(токен + deviceId + minutesUsed = обычно 1)

Сервер:
проверяет, что токен валидный
проверяет, что это устройство (HWID) действительно привязано к этой лицензии
если триал → списывает минуты за сегодня
если превышен лимит на день → возвращает ошибку
обновляет список IP-адресов (для отладки / аналитики)
иногда возвращает новый токен, если тип лицензии поменялся

Оффлайн-режим (когда нет интернета или сервер не отвечает)
Приложение хранит в файле licstate.dat последнее известное состояние:
это был триал → сколько минут оставалось
это была платная подписка → до какого числа действовала
Пока это состояние не истекло по времени — приложение продолжает работать.
Пример сообщений в оффлайне:
«Trial: 18 min (offline mode)»
«Subscription active • remaining 14d 3h 41m (offline mode)»
«Trial expired (offline mode)» → блокировка


Ключевые защиты (что сложно обойти)
HWID привязано к лицензии → нельзя залогиниться с одного устройства под двумя аккаунтами
Максимум 3 устройства за 7 дней (старые вытесняются только если им > 7 дней)
Триал — только 60 минут в сутки (сбрасывается в 00:00 по серверному времени)
Токен проверяется на сервере → подделать сложно (JWT + секретный ключ)
Список IP тоже ведётся → можно увидеть подозрительную активность
Оффлайн-режим ограничен последним реальным ответом сервера


######     Админ-панель JoyControl (ardu.live/admin)     ######
— закрытая зона только для владельца/администраторов:
Что видит и может делать админ:

Главная таблица: все пользователи + их лицензии (тип, статус, дата окончания, остаток триала, IP-адреса, заметки)
Быстрые действия по пользователю:
Активировать 1 год / 2 года / навсегда
Установить кастомную дату окончания
Сбросить на триал (с сохранением HWID)
Обнулить использованные минуты триала сегодня
Вручную изменить остаток минут триала

Управление оптовиками (/admin/joylic):
Список оптовиков
По каждому — таблица всех его кодов (статус: не активировано / активировано кем+когда / заморожено)
Массово: заморозить/разморозить/удалить все коды оптовика
По одному: заморозить/разморозить/удалить код

Создание оптовых кодов (/admin/joyadd):
Выбрать количество (до 100)
Типы: 1 год / 2 года / пожизненно (можно несколько сразу)
Кому: существующий оптовик или новый (ФИО + email)
После создания — список кодов + копирование (все / с типами)


Админ видит, кто где активировал коды, может замораживать остатки, выдавать подписки вручную и полностью контролировать систему лицензий.


######      Обычный пользователь     ###### 

видит в профиле и связанных страницах:
Свои данные: ID, имя, email
Кнопки: Редактировать профиль / Выйти
Ссылки: страховка → управление лицензиями → активировать ключ
Блок личной лицензии: тип (триал/год/2 года/навсегда), статус, дата окончания, остаток минут триала
Если лицензии нет — предложение активировать код
Свои заявки на страховку (с фото чека, статусом «На рассмотрении» / «Одобрено», можно менять фото и удалять до одобрения)

######      Оптовик      ######

видит всё то же + дополнительно:
На странице управления лицензиями — блок «Мои оптовые коды»
Список всех своих купленных кодов: код, тип подписки, статус (не активировано / активировано кем+когда / заморожено)
Кнопка «Скопировать все коды» и копирование по одному
Оптовик не может редактировать/замораживать/удалять свои коды — это только админ.

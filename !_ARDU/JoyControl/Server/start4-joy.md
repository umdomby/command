Как это будет работать с точки зрения администратора

Создание оптовых лицензий
Админ заходит на /admin/joyВ верхней части страницы появляется блок «Create Bulk Licenses»:
Выпадающий список: 0–100 (сколько лицензий создать за раз)
Три чекбокса: Year (1 год), Two Years (2 года), Lifetime (навсегда)
Можно выбрать любые комбинации (например, все три сразу)
Поле ввода имени оптовика (до 300 символов, уникальное)
Если имя новое — создаётся новый оптовик
Если имя уже есть — можно выбрать из выпадающего списка существующих
Кнопка «Create»
После нажатия создаётся ровно столько кодов, сколько указано, по каждому выбранному типу.
Каждый код — это уникальная 32-символьная hex-строка (например a1b2c3d4e5f67890...).
Список оптовиков
Ниже появляется список всех оптовиков с количеством лицензий рядом (например: «MegaShop (342 лицензии)»).Клик по имени оптовика → открывается модальное окно с таблицей его лицензий:
Тип лицензии
32-значный код
Статус (Activated / Not Activated / Frozen)
Кнопки Freeze / Unfreeze и Delete для каждой лицензии отдельно
Вверху модалки:
Кнопка «Freeze All» — заморозить все лицензии этого оптовика
Кнопка «Delete All» — удалить все лицензии этого оптовика
Кнопка «Copy All Codes» — копирует в буфер все коды через перенос строки (удобно для отправки оптовику)

Что происходит с лицензией после создания
Все коды изначально не активированы и не заморожены
Оптовик получает список кодов (админ копирует и отправляет ему)


Как это работает для обычного пользователя
Страница: /joy/code

Пользователь вводит 32-значный код в поле.
Система проверяет:
существует ли код
не активирован ли уже
не заморожен ли

Если всё хорошо → лицензия активируется у текущего пользователя:
создаётся/обновляется запись в joyLicense
тип подписки берётся из BulkLicense (year / two_years / lifetime)
дата окончания рассчитывается автоматически (или null для lifetime)
в таблице BulkLicense код помечается как activated + activatedById + activatedAt

После этого код больше нельзя использовать (даже другой человек).
Защита от подбора:
5 неверных попыток подряд → блокировка ввода на 1 час
счётчик попыток хранится в localStorage браузера
через час блокировка снимается автоматически, попытки обнуляются


Кратко: жизненный цикл кода

Админ создал → код в БД, не активирован
Оптовик передал код пользователю
Пользователь ввёл на /joy/code → если валидно → активировал себе подписку
Код стал неактивным навсегда
Админ видит в интерфейсе, что код активирован (зелёный статус)

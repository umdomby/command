Один из моих проектов, как пример
мобильный робот на базе микроконтроллеров ESP8266 или ESP32-S3 с акцентом на удалённое управление и передачу видео в реальном времени.
Вот что он умеет / основные возможности (на основе структуры репозиториев и типичных реализаций таких проектов):
Основные функции управления

Движение вперёд / назад / влево / вправо / повороты на месте
Регулировка скорости (PWM)
Поддержка разных режимов (плавный разгон/торможение, резкие команды)
Управление через веб-интерфейс (Next.js 16 на https://ardua.site/)
Управление через десктоп-приложение на C# (.NET 9)
Джойстик / виртуальные кнопки / WASD / геймпад в веб- и C#-клиенте

Видео и камера

Передача видео с робота в реальном времени через WebRTC (низкая задержка)
Есть отдельный Android-клиент, который специализируется именно на приёме видео по WebRTC
Веб-клиент (ardua.site) тоже показывает видео
WebRTC-сервер + TURN-сервер для проброса через NAT (чтобы видео работало из любого места в интернете)
NGINX как reverse proxy / для HTTPS и правильной маршрутизации

Архитектура и связь

ESP32-S3, ESP8266 + L298N и более мощные драйвера (классический дешёвый вариант с мотор-драйвером на 2–4 DC-мотора)
Связь идёт через WebSocket (server.ts — это socket-сервер на Node.js / TypeScript)
Клиенты (браузер / C# / Android) → websocket-сервер → Wi-Fi → ESP
Команды передаются в текстовом / JSON-формате (типа "F100", "L", "STOP", "SERVO:90" и т.п.) и бинарном виде

Всё задеплоено в Docker-контейнерах (очень удобно для домашнего сервера / VPS) - установлен домашний сервер Ubuntu

Дополнительные возможности
Управление светом / LED / фарой
Управление сервоприводом (поворот камеры вверх-вниз / влево-вправо)
Мультипользовательский доступ, доступ по созданной ссылке

Кратко: что умеет робот
Ездить по командам из любой точки мира (через интернет)
Показывать живое видео с камеры (WebRTC, низкая задержка)
Управляться с телефона (Android-приложение), компьютера (C# программа) или браузера (ardua.site)
Работать через домашний Wi-Fi или мобильный интернет (благодаря TURN и NGINX)

По сути — это классический FPV-робот / танк / шасси с камерой для экспериментов, обучения, слежки за домом, игр или просто хобби.

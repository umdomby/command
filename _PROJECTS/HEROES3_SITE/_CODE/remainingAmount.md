remainingAmount — это переменная, которая обычно используется для отслеживания оставшейся суммы ставки, которую можно распределить 
между участниками, поставившими на противоположного игрока. В контексте вашего кода, remainingAmount представляет собой часть вашей ставки, 
которая ещё не была использована для покрытия прибыли (profit) других участников.
Как используется remainingAmount:
Инициализация: Перед началом цикла remainingAmount должен быть инициализирован значением, равным вашей полной ставке или её части,
которую вы хотите распределить. Цикл распределения: В процессе цикла remainingAmount уменьшается на величину, 
которая используется для покрытия profit участников, поставивших на противоположного игрока. Это происходит в строке:
```
remainingAmount -= overlap / (currentOdds - 1);
```
Здесь overlap — это часть вашей ставки, которая используется для покрытия прибыли другого участника.
Проверка завершения: Цикл продолжается до тех пор, пока remainingAmount больше нуля и есть участники, чья прибыль ещё не полностью покрыта.
Остаток: Если после завершения цикла остаётся ненулевое значение remainingAmount, это означает, что не вся ваша ставка была использована 
для покрытия прибыли других участников. Этот остаток может быть сохранён в переменной overlapRemain или использован другим образом.
Пример использования:
Предположим, вы сделали ставку в размере 100 единиц, и remainingAmount изначально равен 100. 
В процессе распределения вы покрываете прибыль других участников, и remainingAmount уменьшается. 
Если после покрытия всех возможных прибылей остаётся, например, 20 единиц, это значение может быть сохранено как остаток вашей ставки.
Если remainingAmount не инициализирован или неправильно обновляется, это может привести к тому, что ваша ставка не будет корректно 
распределена между участниками. Убедитесь, что remainingAmount правильно инициализирован и обновляется в процессе выполнения кода.

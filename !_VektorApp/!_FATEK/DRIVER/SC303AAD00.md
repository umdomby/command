FA008 = PGCL1 - PGCL12 - PGCL13 - PGCL14 - PGCL15  -зафиксировалось только done на 5 - это нормально?
Коды PGCL (от Pulse Generator Clear) означают этапы очистки различных типов данных в памяти энкодера:
PGCL1 – PGCL13: Очистка логов ошибок, данных о перегрузках и внутренних счетчиков.
PGCL14 – PGCL15: Очистка данных о многооборотных значениях (Multi-turn data) — это самый важный этап для устранения ошибки .810.
Нажми F несколько раз → зайди в Utility Function (FAxxx)
Выбери FA008 (Reset Absolute Encoder)

# .941 RESTART

Pn002 = 0100 — драйвер работает в инкрементальном режиме (самый стабильный).
Pn002 = 0200 — переводим в однооборотный absolute.
Pn002 = 0000 — заводской режим (обычно многоборотный absolute, если энкодер поддерживает).

Pn013 = 0001 (Station Address).
Pn014 = 0000 (использовать Pn013 как адрес).

Pn00B = 1010: Значение 10XX переводит драйвер в автономный режим позиционирования (Internal Positioning), который не требует связи по сети. 
Это лучший способ проверить, жив ли драйвер и двигатель.
Pn001 = 1000: Этот параметр управляет входом S-ON (Servo On). Значение 100X позволяет "включить" сервопривод, замкнув контакт CN1-9 на 0 В, 
либо сразу включает его при подаче питания.

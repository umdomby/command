ИИ модель обнаружение объектов есть 
Classification      Определяет, что на картинке целиком (один класс).	«На фото кошка».
Object Detection    Находит объекты и рисует вокруг них рамки (Bounding Boxes). «Тут кошка в левом углу, а тут собака в правом».
Segmentation        Выделяет контуры объекта попиксельно (Masks). «Вот точные границы кроны дерева и забора».
Pose Estimation     Находит «скелет» объекта (ключевые точки/Keypoints). «Поза человека: руки подняты, колени согнуты».
OBB (Oriented Detection) Обнаружение повернутых объектов (удобно для аэросъемки). «Корабль развернут под углом 45 градусов».
Track               Отслеживает перемещение объекта на видео.«ID 1 (машина) едет прямо, ID 2 (пешеход) переходит дорогу».


Структура папок (YOLO standard)
/my_dataset
│
├── data.yaml          # Главный файл: пути к папкам и названия классов
├── train/             # Данные для обучения
│   ├── images/        # Картинки (image1.jpg, image2.jpg...)
│   └── labels/        # Текстовые файлы с разметкой (image1.txt...)
│
├── val/               # Данные для проверки (валидации)
│   ├── images/
│   └── labels/
│
└── test/              # (Опционально) Финальный тест
├── images/
└── labels/
Формат Label-файла (для Detection):
Внутри текстового файла для каждой картинки лежат строки:
<class_id> <x_center> <y_center> <width> <height> (все значения от 0 до 1).


У этой стратегии есть три основных названия в зависимости от контекста:ROI (Region of Interest) Pooling / Processing: 
Выделение «Области интереса». Вместо того чтобы скармливать нейросети весь кадр $1920 \times 1080$, вы вырезаете только нужный кусок 
(например, конвейерную ленту или дверной проем) и работаете только с ним.Static Background Masking: Маскирование статичного фона. 
Если камера не двигается, всё, что вне вашей зоны, — это «белый шум», который мы игнорируем.Anomaly Detection (One-Class Classification): 
Когда вы обучаете модель только на «нормальных» кадрах, чтобы она сигнализировала, если в кадре появилось что-то «не нормальное» (аномалия)

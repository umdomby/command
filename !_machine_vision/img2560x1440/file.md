координаты в .txt файле?
Это нормализованные координаты bounding box в формате YOLO:

class_id — номер класса (0 = fallen, 1 = normal_bottle)
center_x — центр прямоугольника по ширине (от 0.0 до 1.0)
center_y — центр прямоугольника по высоте (от 0.0 до 1.0)
width — ширина прямоугольника в долях от ширины изображения
height — высота прямоугольника в долях от высоты изображения

Пример:
0 0.6234 0.4512 0.1847 0.3125

ТОЛЬКО упавшие бутылки (класс fallen = 0).
Это нормально и даже предпочтительно на старте.
Не нужно помечать все нормальные бутылки (normal_bottle).
Модель отлично научится находить "аномалию" (упавшую), если будет много примеров только с классом fallen.



Как должны выглядеть файлы разметки (.txt)
Для каждого изображения photo001.jpg должен быть файл photo001.txt в папке labels/train/ (или val).
Содержимое одного .txt файла (по одной строке на каждый объект):
textclass_id center_x center_y width height
Пример (если упала одна бутылка):
text0 0.512 0.623 0.184 0.317

Все значения нормализованные от 0.0 до 1.0 (относительно ширины и высоты изображения).
class_id = 0 — потому что у нас только один класс (fallen).

Как быстро подготовить такую структуру

Создай папку fallen_bottle_dataset.
Внутри создай папки images/train, images/val, labels/train, labels/val.
Разложи свои фото (2500×1400) по images/train и images/val.
Размечай bounding boxes (красным прямоугольником с запасом 3–6 пикселей) вокруг каждой упавшей бутылки/пятна.
Сохраняй разметку в формате YOLO .txt (это делают LabelImg, Roboflow, CVAT и т.д.).
Создай файл data.yaml как выше.

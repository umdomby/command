подготовить датасет для YOLO (обучение на упавших бутылках)
Ты используешь OpenCvSharp + RTSP 2560×1440 @20 fps.
YOLO (рекомендую YOLOv8n или YOLOv8s) будет работать отлично, но сначала нужен качественный датасет.
1. Как создать картинки (кадры)

Запусти свой RTSP-поток (VLC → медиа → поток → rtsp://127.0.0.1:8554/mystream).
Извлеки кадры ffmpeg (самый удобный способ):

Bash# 5 кадров в секунду, качество 95%, чтобы не было артефактов
ffmpeg -i rtsp://127.0.0.1:8554/mystream -vf fps=5 -q:v 2 frames/%06d.jpg
Собери 800–1500 кадров для train + 150–300 для val.
Делай разнообразие:

бутылки в разных положениях (вертикально, 30°, 45°, 90° — лежат)
разные скорости конвейера
разные освещения (включать/выключать свет, блики)
бутылки с водой, пустые, разных цветов (если нужно)
ложные срабатывания: тени, блики, отражения, перекрытия

3. Разметка (какой класс где)
   Нормально (upright):

бутылка стоит вертикально или наклон < 20–25°
высота bounding box >> ширины (aspect ratio > 2.0)

Тревога (fallen):

бутылка лежит на боку (угол > 45°)
высота << ширины (aspect ratio < 1.0)
бутылка перевернута, лежит поперёк конвейера, перекрывает соседние

Важно: размечай только бутылки на конвейере. Края кадра, фон, руки оператора — не размечай.
Инструменты для разметки (бесплатные):

LabelImg (самый простой)
Roboflow (рекомендую — можно сразу экспортировать в YOLOv8)
LabelStudio

4. Размеры изображений и модели

Исходные кадры — оставляй 2560×1440 (YOLO сам ресайзит).
Рекомендуемый размер обучения:
imgsz=1280 — лучший баланс качества/скорости для твоего разрешения
imgsz=640 — если хочешь максимальную скорость (на слабом ПК)

В приложении (C#) перед детекцией делай ресайз до 1280×720 или 640×640 — это сильно ускорит.

5. Логика оповещения + защита от ложных срабатываний
   Самая частая проблема — ложные тревоги на бликах/тенях.
   Рекомендуемая логика (в коде после YOLO):
   C#// после получения результатов детекции
   foreach (var detection in results)
   {
   if (detection.Confidence < 0.70) continue;           // высокий порог
   if (detection.Class != "fallen") continue;

   // доп. проверка по геометрии (даже если модель ошиблась)
   double aspect = (double)detection.BBox.Height / detection.BBox.Width;
   if (aspect > 1.2) continue; // всё ещё слишком "высокая" — не упала

   // временная стабильность (3–5 кадров подряд)
   fallenCounter++;
   if (fallenCounter >= 4)
   {
   TriggerAlarm(); // звук + MessageBox + лог + остановка конвейера
   fallenCounter = 0;
   }
   }
   Дополнительная защита от ложных:

ROI (регион интереса) — рисуй прямоугольник только над конвейером (например, x=300..2200, y=400..1100). Детекцию делай только внутри.
Temporal filter — упавшая бутылка должна держаться 4+ кадров.
NMS — YOLOv8 уже делает, но можно усилить iou_threshold=0.45.
В датасет специально добавь 100–150 кадров с ложными объектами (тени, блики) и размечай их как upright или вообще не размечай.
Используй YOLOv8s вместо nano — она точнее на сложных сценах.
После обучения прогони на 200 тестовых кадрах и подкрути confidence.

6. Как дальше интегрировать в твой проект (коротко)

Обучи в Python (ultralytics):Bashyolo train model=yolov8s.pt data=data.yaml epochs=100 imgsz=1280
Экспортируй: yolo export model=best.pt format=onnx
В C# добавь NuGet:
Sdcb.Yolo (самый простой для .NET 8) или
Microsoft.ML.OnnxRuntime + OpenCvSharp DNN

В цикле чтения кадров:C#var resized = frame.Resize(new Size(1280, 720));
var results = yolo.Run(resized); // твоя модель
// рисуй боксы + логика тревоги выше
